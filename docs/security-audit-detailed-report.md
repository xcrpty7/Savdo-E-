## 🔍 Audit Dimensions

### 1. Authentication & Session Management
- **JWT Implementation**: 
  - ✅ Uses short-lived access tokens and long-lived refresh tokens.
  - ✅ Implements **Token Rotation**: Refresh tokens are revoked upon use, preventing replay attacks.
  - ✅ Secure cookie flags (`httpOnly`, `secure`, `sameSite: 'strict'`) are used in production.
- **Password Security**:
  - ✅ Strong hashing with `bcryptjs` (salt rounds = 10).
- **Session Control**:
  - ✅ `logoutAll` functionality allows users to invalidate all active sessions.

### 2. Authorization & Access Control
- **RBAC (Role-Based Access Control)**:
  - ✅ Middleware `authorize(...roles)` strictly enforces access based on user roles (`ADMIN`, `SUPER_ADMIN`).
- **IDOR Protection**:
  - ✅ Object-level authorization is implemented in services. For example, `getUserOrders` and `getOrderById` verify that the requesting user owns the resource unless they have administrative privileges.

### 3. Input Validation & API Security
- **Validation**:
  - ✅ All critical endpoints (Auth, Orders, Products) use `Joi` schemas for strict input validation.
- **Rate Limiting**:
  - ✅ `express-rate-limit` is applied globally and specifically for authentication routes to mitigate Brute-Force and DoS attacks.
- **HTTP Hardening**:
  - ✅ `helmet` is used to set secure HTTP headers.
  - ✅ CORS is restricted to a specific allowlist of origins.
  - ✅ Body size limit (50kb) is enforced to prevent payload-based DoS.

### 4. AI & LLM Security
- **Prompt Injection**:
  - ❌ **Finding**: The `ai.controller.js` was vulnerable to prompt injection because user input was concatenated directly into the system prompt.
  - ✅ **Remediation**: Implemented a structured system prompt with explicit "System Constraints" and delimiters (`---`) to separate instructions from user input. This prevents the AI from being manipulated into ignoring its role.

### 5. Frontend-Backend Integration
- **Admin Panel**: ✅ Correctly handles token refresh and authorization headers.
- **Web Store**: ✅ Implemented robust `axios` interceptors for automatic token refreshing and global error handling.
- **Mobile App**: ✅ Securely manages tokens in store and handles 401 redirects to the Login screen.
